# Baton — 座艙外觀(`TUI.yaml`)

[English](TUI.md) · **繁體中文**

座艙從 `$HOME/.baton/TUI.yaml` 讀取它的**外觀**:色彩**主題**與群組分割的**版面配置**。它是與主
`config` 分開的檔案,讓你能重塑外觀而不動到你的按鍵綁定與行為設定。伺服器讀取它、把它合併進生效的設定,
再廣播給每個前端,所以一個 `C-t R`(或對常駐程式送 `SIGHUP`)就能熱重載它,不需重啟整支隊伍。這個檔案是
**選用的**——缺少或不完整的 `TUI.yaml` 會讓內建外觀維持原樣。

```yaml
# $HOME/.baton/TUI.yaml
theme:
  brand: "33" # primary accent — banner, active borders, selection
  brand-hi: "117" # brighter accent — titles, pins, the summary tile, search hits
  attention: "#ff5f5f" # the attention-state LED

default-layout: main-vertical # the layout a group opens with

layouts:
  - name: review # a custom layout, selectable alongside the presets
    areas:
      - [diff, diff, log]
      - [diff, diff, sh]

scratch: # the floating scratch pane (C-t ~)
  command: "" # program to run; empty = the default shell
  width: 0.8 # box size as a fraction of the terminal
  height: 0.6
```

## 主題

每個 token 都是一個色彩字串——ANSI 256 索引(`"33"`)或十六進位 `"#rrggbb"`。**空的或缺少的 token 會保留它的
內建預設值**,所以部分主題只會改動它所指名的部分。未知的色彩字串會以終端機預設色呈現,而不會把座艙卡死。

| Token       | 色彩                             |
| ----------- | -------------------------------- |
| `brand`     | 橫幅、作用中的磚／卡片邊框、選取 |
| `brand-hi`  | 標題、釘選符號、摘要磚、搜尋命中 |
| `spawning`  | `spawning` 狀態的 LED            |
| `running`   | `running` 狀態的 LED             |
| `idle`      | `idle` 狀態的 LED                |
| `attention` | `attention` 狀態的 LED           |
| `exited`    | `exited` 狀態的 LED              |

## 版面配置

一個群組的**分割**(見 [SPEC.zh-TW.md](./SPEC.zh-TW.md#工作項目work-items))負責排列它的即時磚。預設值——`tiled`——是均勻
網格。在分割中按 `L` 會讓排列方式在內建預設與你定義的任何自訂版面配置之間**輪替**;這個選擇是
**伺服器持有、以群組為單位**,會隨著快照跟在可見數量旁邊,並且**跨重啟保存**。搭配
[巢狀群組](./SPEC.zh-TW.md#工作項目work-items),子群組會顯示為一個 `▣` 匯總磚——尺寸像即時面板一樣——列出它所含成員／
子子群組的一小份名冊,每個都帶一個狀態 LED,外加一個 `↵ descend` 提示(溢位的 `▦` 摘要磚帶有同樣的名冊 +
`↵ open`)。`enter` 會**下降**進去(標頭以麵包屑顯示路徑,`backend › api`);`esc` / `b` 退回上一層。子群組磚上的
`⊙` 標記一個**釘選預設**——它其中恰好一個直屬面板被釘選——此時下降會直接落進那個面板的放大,而不是子群組的
分割,和頂層群組從儀表板享有的同一個單一釘選捷徑一樣(**返回**,`C-t b`,退回到子群組的分割)。那個分割的 `⊙`
不是儀表板的**最愛**:`*` 把所選卡片設為最愛——這是一個獨立、伺服器持有的旗標,會把面板或群組浮到網格與樹的
**最前**,而且永不觸及哪些磚以即時串流或單一釘選的下降。每一層都保有自己的版面配置、可見數量與縮放比例,因為
這些全都以群組路徑為鍵。

### 預設

| 名稱              | 排列方式                                    |
| ----------------- | ------------------------------------------- |
| `tiled`           | 均勻網格——每個磚同樣大小(預設)              |
| `main-vertical`   | 第一個成員填滿一個高高的左欄;其餘堆疊在右邊 |
| `main-horizontal` | 第一個成員填滿一個寬寬的頂列;其餘在下方排開 |
| `stack`           | 每個成員都是全寬,由上到下堆疊               |

`default-layout` 指名一個群組在你輪替它之前開啟時所用的版面配置(空 = `tiled`)。

### 自訂版面配置

自訂版面配置在一個 `areas` 網格中為各區域命名——每行一列區域名稱,每格指名擁有它的區域,所以一個橫跨數格的
區域會重複它的名字(即 CSS grid-template-areas 模型)。成員依閱讀順序(以列為主、依首次出現)填入各區域;`.`
標記一個空格。

```yaml
layouts:
  - name: review
    areas:
      - [main, main, side]
      - [main, main, side]
      - [main, main, foot]
```

這裡 `main` 橫跨左邊一個 3×2 的區塊,`side` 佔右欄上方兩格,`foot` 佔右下角那格——所以第一個成員拿到大面板,
接下來兩個拿到堆疊的側邊面板。一個區域**必須是實心的矩形**;非矩形、此前端未知、或對終端機來說太小的自訂版面
配置會**退回到均勻網格**,所以只存在於某一個前端設定裡的版面配置永遠不會把分割卡死。超過區域數量的成員會收摺
進**摘要磚**,和它們在均勻網格裡的做法完全一樣。

### 縮放

在分割中按 `z` 進入**縮放模式**:方向鍵(或 `h` / `j` / `k` / `l`)放大與縮小聚焦的磚,`tab` 把焦點移到另一個磚,
`z` / `esc` 完成。這個縮放會扭斜目前版面配置的列與欄——所以它適用於**除了**均勻 `tiled` 網格以外的每個版面配置,
因為 `tiled` 沒有可調的逐軌尺寸(先按 `L` 換一個分割版面配置)。任何會把某個磚縮到小到無法呈現的微調都會被拒絕,
所以分割永不會在縮放途中彈回均勻網格。縮放是**檢視本地的**:它只活在這個座艙裡(永不送到伺服器),持續到你輪替
版面配置或離開群組為止,並且**在重新接上時重置**——這與伺服器持有的版面配置和可見數量不同。

## 暫存面板(scratch pane)

`C-t ~` 會浮出一個**暫存面板**——一個用完即丟的 shell(或任何 `command`)——覆在你當下所在的任何檢視之上,就是
tmux 的 `display-popup`,讓你快速跑個 `git`／`ls`／`htop` 而不必離開隊伍。它是伺服器端的**短暫** PTY:它永不加入
隊伍、儀表板或持久化狀態,而且會在你關閉它或座艙斷線時被回收。在它裡面,每個按鍵都驅動 shell;領導鍵是唯一的
逃脫路徑——`C-t ~` **隱藏**它(shell 繼續執行,所以重新開啟會從你離開處接續),`C-t w` 把它**永久關閉**,而
`C-t C-t` 送出一個字面的前綴。這個框以設定的 `width`／`height` 比例(預設 `0.8`×`0.6`)置中於終端機,並以一個
易讀的最小值為下限,並在終端機改變大小時重新排版。

| 欄位      | 意義                           |
| --------- | ------------------------------ |
| `command` | 面板要跑的程式(空 = shell)     |
| `width`   | 框寬,佔終端機的比例(0 = `0.8`) |
| `height`  | 框高,佔終端機的比例(0 = `0.6`) |

## 相關的座艙按鍵

這些跟外觀設定一起出現(完整按鍵參照見 [SPEC.zh-TW.md](./SPEC.zh-TW.md#按鍵)):

| 位置              | 按鍵      | 作用                                           |
| ----------------- | --------- | ---------------------------------------------- |
| 任何檢視          | `C-t ~`   | 切換浮動的暫存面板(用完即丟的 shell)           |
| 群組分割          | `L`       | 輪替磚的版面配置(預設,再來是你的自訂版面配置） |
|                   | `z`       | 縮放模式——方向鍵放大／縮小聚焦的磚(檢視本地）  |
| 捲動模式(`C-t [`) | `v`       | 開始整行選取                                   |
|                   | `V`       | 開始**區塊**(矩形)選取                         |
|                   | `h` / `l` | 在區塊選取中,把欄邊往內／往外拉                |
| 群組分割(開滑鼠)  | 點擊      | 聚焦游標下的磚(在 `C-t k` 切換滑鼠)            |

見 [PLUGIN.zh-TW.md](./PLUGIN.zh-TW.md#可程式化的標題--paneltitle) 了解 `panel.title` hook,它讓每個面板的標題
本身可以從 Lua 程式化控制。

## 螢幕保護 🟢

讓座艙閒置幾分鐘,它就會滑進一個螢幕保護:一整面全螢幕的數位雨簾幕,中央發光著 **BATON** 字樣與一個大時鐘。
任何按鍵或點擊都會喚醒它——而那個按鍵會被吞掉,所以你回來的路上永遠不會誤動到隊伍。它只會畫在靜止的檢視之上
(絕不會蓋在即時放大、分割、暫存面板或開啟的提示上),而且後端一有打嗝就會立刻把它拉到一旁,所以故障永遠不會
被藏在雨後面。

沒耐心?領導鍵能隨傳隨到——這個按鍵是刻意不放進按鍵對應表的。它只是雨和一個時鐘;它不觸及伺服器上的任何東西。
